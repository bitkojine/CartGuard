This page answers the hard implementation questions investors and technical judges ask.

Where We Store Legal Context

Short answer: versioned policy packs, not hardcoded rules.

  • Store legal rules as machine-readable policy files in a policy pack repo.
  • Each rule has metadata: jurisdiction, source law URL, version, effective date, risk level.
  • Keep extracted guidance snippets and mapping notes with citations.
  • Use semantic versioning so every validation run references a rule version.
  • Output evidence artifacts with rule IDs + source references for audit replay.

Do We Start With One Country?

Yes. Start narrow: one jurisdiction profile first, then expand. For this wedge, begin with EU cosmetics common criteria and one priority market profile before adding more country layers.

Step 1: EU cosmetics baseline rules Step 2: Country-specific overrides Step 3: Multi-country policy packs

Major Technical Challenges (Most Important)

  1. Legal-to-rule translation: turning legal text into clear, testable, machine rules without losing legal meaning.
  2. Evidence quality: proving that sources are strong enough for compliance teams, not just \"a link exists\".
  3. Deterministic outputs: same input + same policy must always return the same result for audit trust.
  4. Low-friction integration: fitting into real CI/CMS/PIM workflows without slowing release teams down.
  5. Versioned policy management: handling rule updates across countries and customer overrides without breaking consistency.

Why These Are Hard

  • Law and marketing language are ambiguous, code is not.
  • Compliance trust depends on evidence quality, not model confidence.
  • Audit use cases need replayable, version-locked results.
  • Teams reject tools that add release friction.
  • Policy drift creates inconsistent enforcement across markets.

Failure Modes To Avoid

  • False confidence: saying \"compliant\" instead of \"risk reduced\".
  • Weak evidence packs that legal teams do not trust.
  • Too many warnings and too few actionable blocks.
  • Manual policy maintenance that does not scale.
  • Slow pipelines that developers bypass.

Most Important Questions To Answer Right Now (Pitch-Critical)

  1. What exact legal baseline are we implementing first?
  2. Who signs off that our rule mapping is acceptable?
  3. What makes our evidence output trusted by compliance teams?
  4. How quickly can a dev team integrate this in CI?
  5. What pilot metric proves value in 2-4 weeks?
