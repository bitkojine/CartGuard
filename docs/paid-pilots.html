Paid Pilots

How we run pilots that convert to annual contracts.

A paid pilot is a short paid project on real listing workflows. We track business metrics, prove value, and convert or stop fast.

CartGuard provides recommendations on missing documentation and evidence only. No legal advice.

What A Paid Pilot Is

  • Short, paid engagement (usually 2-6 weeks).
  • Runs on real customer data and workflow.
  • Has baseline metrics, weekly reviews, and a signed success plan.
  • Ends with one decision: convert, extend once, or stop.

Outcomes We Must Prove

  • Lower missing-document rate before publish.
  • Faster review cycle from draft to listing-ready.
  • Less rework between ops, compliance, and engineering.
  • Clear conversion case tied to budget owner pain.

Current Pilot Focus (Beachhead)

  • Customer: China-based marketplace-first sellers entering Germany.
  • Priority segment: consumer electronics with 500-5,000 SKUs.
  • Main problem: missing EU-required docs causing delays and rework.
  • Pilot target: prove measurable reduction in delays within 2-6 weeks.

Minimal Data Checklist (Daily)

  • Each active pilot has one clear `Next Exec Step` due in 48 hours.
  • `Current Missing Doc Rate` is updated at least once per day.
  • Every customer touchpoint is logged in `Pilot Notes` within same day.
  • If blocker appears, add note type `Risk` and assign owner in note summary.
  • If proof appears, add note type `Evidence` with artifact link.

Team Tasks By Role

  • Engineer: integrate checks, instrument metrics, reduce false positives.
  • Pilot lead: run cadence, unblock decisions, track commitments.
  • Sales: map outcome to budget owner and drive conversion.
  • Research agents: validate claims, source evidence, flag uncertainty.

Pilot Scorecard (Go / No-Go)

  • Go: measurable improvement in at least 2 core metrics and budget-owner commitment to annual plan.
  • Conditional Extend: improvement exists but one hard blocker still open, one short extension only.
  • No-Go: no measurable improvement, unclear owner, or only free-feature demand.
  • Kill immediately if workflow never reaches real production usage.

Current Weaknesses (Keep Brutal)
