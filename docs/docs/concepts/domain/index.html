Domain Concepts and Ubiquitous Language

This glossary is the source of truth for customer-facing and product-facing terms in CartGuard's listing compliance workflow.

Back to Introduction Open Ops Manual

Core Bounded Context: ListingCompliance

Type Term Definition
EntityListingA marketplace listing under review before submission or publish.
EntityProductA sellable item linked to one or more listings and variants.
EntityDocumentA provided artifact such as declaration, test report, or label evidence.
EntityDocumentTypeA required category of document for a listing scenario.
EntityCheckRuleA rule that evaluates whether required document conditions are met.
EntityIssueA missing-document or consistency problem found by checks.
EntityResolutionThe recorded action that closes an issue.
EntityWorkflowStageDraft, InternalReview, SubmissionReady, Submitted, or Live.

Value Objects

Term Definition
MarketplaceChannelChannel identifier, for example Amazon.de.
CountryJurisdiction code used by check rules and reporting.
CategoryProduct category used for requirement selection.
SlaExpected turnaround target for review/resolution steps.
DelayMeasured launch delay attributed to unresolved issues.
RatePercentage metric such as missing-document rate.

Separate Bounded Context: Commercials

Use these terms for sales process and contracts. Keep them out of the core listing-compliance model.

Term Use
AccountCompany record in pipeline and contract workflow.
TrialProjectTime-bounded paid evaluation project.
ContractOngoing commercial agreement after trial.
StakeholderDecision maker, user, or blocker in account workflow.
TrialConversionRateCommercial metric for trial-to-contract outcomes.

Preferred Vocabulary

  • Say: listing, missing document, review delay, rework loop, blocker, resolution.
  • Avoid in buyer copy: wedge, beachhead, kill criteria, stage-safe claim, feature-vs-company test.
  • Use buyer language in UI labels, reports, and error messages.
  • Keep strategy terms in internal pages and planning notes only.

Implementation Guidance

  • TypeScript interfaces in product code should use ListingCompliance terms.
  • Sales/process modules should live under Commercials terms.
  • When adding a new term, update this page before using it in code or copy.
